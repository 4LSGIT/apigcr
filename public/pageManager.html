How landing pages work / AI page prompt

Paste a complete, self-contained HTML file into a page record. A live page is served publicly at /p/<slug>, and — if a host (and optional path) is mapped — at that vanity domain. Form POSTs to the same URL are forwarded to the page's YisraHook and the visitor is redirected to the thank-you target. Use Create hook on a page to bootstrap a hook (spam-filtered, lead fields mapped), then wire its targets in Automation → Hooks.

AI prompt template 

Create a single, self-contained HTML5 landing page file. Requirements:
- One complete HTML document. All CSS and JavaScript inline. No external
  resources (no CDN scripts, no web fonts — use a system font stack).
  Images only by absolute URL if provided.
- Mobile responsive.
- Lead capture form: <form method="post"> with NO action attribute.
  The server handles submission and redirects — do not add any JavaScript
  form handling, fetch, or onsubmit logic.
- Use these exact input names for fields you include: `name`, `email`, `phone`.
- Include this honeypot exactly, inside the form:
  <div style="position:absolute;left:-9999px" aria-hidden="true"><label>Website<input type="text" name="website" autocomplete="off" tabindex="-1"></label></div>
- If the page URL contains `?submitted=1`, show a thank-you message instead of
  (or above) the form. A few lines of inline JS reading location.search is fine
  for this one purpose.
- Page purpose / copy / branding: [DESCRIBE YOUR OFFER, AUDIENCE, AND CTA HERE]
