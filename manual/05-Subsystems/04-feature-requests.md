# Feature Requests — Overview

The Feature Requests board is where staff ask for new features and report bugs
in YisraCase — and where you can see what's already been asked for, vote it up,
and follow its progress. It replaces scattered emails and hallway mentions with
one tracked list IT actually works from.

## What you can do with it

- **Submit a request** — a feature idea or a bug report, with a title and a
  description.
- **Vote** — upvote requests you also want, so the most-wanted ones rise to the
  top. Vote again to remove your vote.
- **Comment** — discuss a request or reply to someone else's comment.
- **Follow progress** — each request moves through stages (Considering →
  Planning → Working on it → Implemented), so you can see where your idea stands.

## Where to find it

**More → Feature Requests** in the main navigation. Any logged-in user can use it.

---

## Submitting a request

1. Click the new-request button and fill in:
   - **Type** — ✨ Feature or 🐛 Bug.
   - **Title** — a short summary ("Add a print button to the case page").
   - **Description** — the details: what you want, or — for a bug — what
     happened, what you expected, and how to reproduce it.
2. Submit. IT is notified by email automatically, and your request appears on
   the board for others to see and vote on.

## Voting and commenting

- The vote button on each request toggles your vote on and off; the count shows
  how many people want it.
- Open a request to read and add comments, including replies to existing
  comments — useful for clarifying a request or adding a "me too, and also…".

## Stages (who sets them)

Each request carries a **stage** that shows its status:

- **Considering** — logged, not yet decided.
- **Planning** — accepted, being scoped.
- **Working on it** — actively in progress.
- **Implemented ✅** — done and shipped.

Anyone can see the stage; **only admins/IT change it**, along with an optional
status note and a progress indicator. So the board doubles as a lightweight
roadmap.

---

## A few things to know

### Vote before you re-submit

Before adding a new request, skim the board — if it's already there, vote and
comment instead of creating a duplicate. It keeps the most-wanted items clearly
ranked.

### Bugs go here too

Despite the name, this is also the bug tracker. Tag it 🐛 and give repro steps;
it lands in the same place IT watches.

### IT gets notified automatically

New requests email IT, so you don't need to also send a separate message — the
submission is the notification.
